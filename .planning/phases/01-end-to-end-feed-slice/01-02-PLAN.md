<objective>
Build the ingestion slice on the VERIFIED FRBP rulemaking source. First, identify and verify the real authoritative FRBP rulemaking source (a `[BLOCKING]` task — before any fixture or seed work). Then build a `SourceAdapter` seam with one adapter that fetches that source, resolves every fetch to an explicit tri-state outcome (CHANGED / UNCHANGED / FETCH_FAILED), and an append-only snapshot store that persists a versioned snapshot on every CHANGED fetch. Seed the registry with the one national FRBP source (D-01). Capture a real live snapshot pair for the fixture-replay validation strategy (D-03).

Context — what changed from the reviewed plan: All four cross-AI reviewers returned DO-NOT-SHIP on a single unanimous finding — the previous plan seeded the generic `uscourts.gov/news/rss` Judiciary News feed while the declared subject layer is FRBP. The user's resolved product decision is: **find a real FRBP-specific source.** D-01 stays locked (the layer is FRBP). D-02's understanding is updated: the specific source is no longer "the news feed" — it is now "the verified FRBP rulemaking source", the U.S. Courts Pending Rules & Forms Amendments / rulemaking area. RESEARCH.md established that the rulemaking page itself carries no RSS/Atom feed. The `SourceAdapter` seam abstracts fetch strategy: if the verified FRBP source has no clean feed, this source's adapter fetches the rulemaking HTML page directly with a thin HTML-list parse — that is a property of THIS one source's adapter, not the general per-source HTML-scraping framework (which remains Phase 2). Task 1 makes that feed-vs-page determination from live evidence, not assumption.

Purpose: This is the slice that gets real FRBP-source content into the system. After this plan a developer can run the fetcher against the seeded source (live or fixture-replayed) and observe a snapshot row being stored — the input to all downstream detection.
Output: Verified-source record, tri-state fetcher, append-only snapshot store, seeded FRBP source, captured live-snapshot fixtures, passing adapter + snapshot tests.
</objective>

<objective>
Build the ingestion slice: a `SourceAdapter` seam with one `RssAdapter` implementation that fetches the uscourts.gov Judiciary News RSS feed, resolves every fetch to an explicit tri-state outcome (CHANGED / UNCHANGED / FETCH_FAILED), and an append-only snapshot store that persists a versioned snapshot on every CHANGED fetch. Seed the registry with the one national FRBP source (D-01, D-02). Capture the fixture feed pair for the fixture-replay validation strategy (D-03).

Purpose: This is the slice that gets real source content into the system. After this plan a developer can run the fetcher against the seeded source (live or fixture-replayed) and observe a snapshot row being stored — the input to all downstream detection.
Output: Tri-state RSS fetcher, append-only snapshot store, seeded FRBP source, captured feed fixtures, passing adapter + snapshot tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uscourts.gov feed → fetcher | Untrusted external XML content crosses here |
| fetcher → snapshot store | Parsed/normalized content persisted to Postgres |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service / Information Disclosure | `feedparser.parse` on malicious/malformed feed XML (XXE, entity expansion) | mitigate | `feedparser` disables external entity resolution by default; oversized/empty bodies and unparseable feeds resolve to FETCH_FAILED (Pattern 2); `httpx` timeout of 30s caps slow-loris payloads |
| T-02-02 | Spoofing / Tampering | A silent fetch failure read as "no change" | mitigate | Exhaustive tri-state outcome — every code path provably yields CHANGED/UNCHANGED/FETCH_FAILED; unit tests cover timeout, non-200, empty body, unparseable feed (RESEARCH Pitfall 2) |
| T-02-03 | Tampering | Server-Side Request Forgery via the feed URL | accept | The feed URL is a curated, code-seeded constant (`uscourts.gov/news/rss`) — not user-supplied; arbitrary-URL monitoring is explicitly out of scope (REQUIREMENTS.md). Revisit when config-driven onboarding lands in Phase 2 (SRC-06) |
| T-02-04 | Information Disclosure | `httpx` following redirects to an unintended host | accept | `follow_redirects=True` is required (the documented feed aliases 301→`/news/rss`); the seed URL is trusted; low risk for a single curated government source |
</threat_model>

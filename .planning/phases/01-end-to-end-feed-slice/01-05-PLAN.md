<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → review API | Reviewer input (effective date, edited summary, reviewer name) crosses here |
| "the other product" → pull API | The only integration seam; machine caller crosses here |
| API responses → review SPA | Feed/AI text rendered in the browser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | Pull API leaking non-verified changes to "the other product" | mitigate | `GET /changes` filters `status == verified` at the query level; `test_only_verified` integration test enforces it; no write endpoint and no DB credentials exposed (API-07) |
| T-05-02 | Tampering | SQL injection via reviewer input or any filter parameter | mitigate | All endpoints use SQLAlchemy 2.0 parameterized ORM queries; request bodies validated by Pydantic models; no string-formatted SQL |
| T-05-03 | Tampering | Stored XSS — feed/diff/AI summary text rendered in the SPA | mitigate | React escapes JSX by default; `web/src/` uses no `dangerouslySetInnerHTML`; `react-diff-viewer-continued` renders text safely |
| T-05-04 | Tampering | CSRF on approve/edit/reject mutations | accept | Phase 1 review UI is an unauthenticated internal tool with no cookie-based session (explicit Claude's-discretion decision; full auth is Phase 8 / AUDIT-02). No ambient credential exists for a CSRF attack to ride; documented as accepted for the internal-only Phase 1 surface |
| T-05-05 | Spoofing / Repudiation | Reviewer identity is a free-text field, not authenticated | accept | Per CONTEXT.md Claude's-discretion and Deferred Ideas — full reviewer attribution / audit trail is explicitly Phase 8 (AUDIT-02). Phase 1 captures `reviewer_name` as a free-text field only; accepted for an internal tool |
| T-05-06 | Tampering | Illegal lifecycle transition via a crafted API call | mitigate | approve/edit/reject call `assert_transition`; `IllegalTransitionError` maps to HTTP 409 — illegal transitions are rejected, not silently applied |
</threat_model>

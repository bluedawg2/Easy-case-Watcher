<objective>
Build the AI slice: a single-turn Anthropic Messages call that turns a `detected` Change's verbatim diff into the D-05 structured summary (headline + What changed / Where / To whom / For what cases), persists it as JSONB (D-09), attaches the server-set "not legal advice" label (D-08), and advances the Change to `processed`. The prompt enforces concise output (D-06) and the no-speculation / no-advice guardrails (D-07).

Purpose: This is the slice that makes a change human-readable. After this plan a developer can run a `detected` Change through the summarizer and observe a `processed` Change with a structured summary — the content the reviewer reads in plan 01-05.
Output: `ChangeSummary` Pydantic schema, the `summarize` wrapper using structured outputs, pipeline `run_summarize` step, AI tests with a mocked SDK.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| diff text → Anthropic API | Verbatim untrusted feed-derived text is sent to the LLM |
| Anthropic response → database | Model output persisted as JSONB on the Change |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Prompt injection — untrusted feed/diff text instructing the model | mitigate | The diff is passed inside a clearly-labelled user-content data block, never as instructions; the fixed `SYSTEM_PROMPT` pins the task; output is schema-constrained by `messages.parse(output_format=ChangeSummary)` so injected free-form instructions cannot reshape the response; the summary is human-reviewed (plan 01-05) before it can reach the pull API. Full prompt-injection hardening is a later-phase concern; Phase 1 notes the surface and applies these baseline controls |
| T-04-02 | Spoofing | The model omitting or altering the "not legal advice" label | mitigate | The label is a server-side module constant attached in `run_summarize`; it is NOT a `ChangeSummary` field — the model cannot omit or reword it (RESEARCH Pitfall 3; D-08; AI-06) |
| T-04-03 | Information Disclosure | `ANTHROPIC_API_KEY` leaking into source or logs | mitigate | Key loaded via `pydantic-settings` from a git-ignored `.env`; never logged; tests mock the SDK and run without the key |
</threat_model>

<objective>
Build the change-detection slice: a content-hash gate that prevents an unchanged poll from triggering downstream work, and a `difflib` unified-diff step that turns a genuine change into a `detected` Change record with a verbatim diff. Wire the fetch → snapshot → detect path into one `run_ingest` orchestration function used identically by the fixture-replay path (D-03) and the live polling path, with no production-only branch (D-04).

Context — what changed from the reviewed plan: two review findings touch this plan. (1) First-fetch ambiguity (#10): the previous plan's action text and its own test contradicted each other on whether the first snapshot creates a Change — this plan now defines explicitly that the first-ever snapshot establishes a silent baseline and creates NO Change, and the plan text and tests agree. (2) Grep-based test policing (Major Concern): the previous "grep for `if fixture`/`if test`" check is cargo-cult — an implementation can diverge via config flags, DI, or env vars while staying grep-clean. It is replaced by `test_identical_code_path`, a behavioral assertion that the fixture and live entries produce byte-identical detection results for the same bytes.

Purpose: This is the slice that decides whether anything happened. After this plan a developer can replay the captured fixture pair and observe a `detected` Change row with a diff appearing — or run the same code against the live source. No AI, no UI yet.
Output: Hash-gate detector, unified-diff helper, the `run_ingest` pipeline orchestrator, fixture-replay test proving the no-branch guarantee behaviorally.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | A fetch failure mis-detected as "no change" so a real change is silently missed | mitigate | `run_ingest` branches explicitly on `FETCH_FAILED` and returns without claiming no-change; `source.health_status` is set to `failed`; the fixture-replay test asserts the FETCH_FAILED path creates no Change but is not silent |
| T-03-02 | Tampering | A production-only / synthetic-injection branch corrupting the detection guarantee | mitigate | `test_identical_code_path` asserts the fixture and live entries produce byte-identical detection results — a behavioral proof that there is no production-only branch (D-04); fixtures are ordinary stored snapshots / replayed HTTP responses |
| T-03-03 | Denial of Service | A pathologically large diff consuming memory | accept | `difflib` is stdlib and bounded by snapshot size; one curated low-volume source in Phase 1 makes this negligible; revisit if multi-source volume grows |
</threat_model>

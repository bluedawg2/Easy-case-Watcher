<objective>
Build the change-detection slice: a content-hash gate that prevents an unchanged feed poll from triggering downstream work, and a `difflib` unified-diff step that turns a genuine change into a `detected` Change record with a verbatim diff. Wire the fetch → snapshot → detect path into one `run_ingest` orchestration function used identically by the fixture-replay path (D-03) and the live polling path, with no production-only branch (D-04).

Purpose: This is the slice that decides whether anything happened. After this plan a developer can replay the captured fixture pair and observe a `detected` Change row with a diff appearing — or run the same code against the live feed. No AI, no UI yet.
Output: Hash-gate detector, unified-diff helper, the `run_ingest` pipeline orchestrator, fixture-replay test proving the no-branch guarantee.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| snapshot content → detector | Untrusted normalized feed text is diffed and stored on a Change |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | A fetch failure mis-detected as "no change" so a real change is silently missed | mitigate | `run_ingest` branches explicitly on `FETCH_FAILED` and returns without claiming no-change; `source.health_status` is updated; the fixture-replay test asserts the FETCH_FAILED path creates no Change but is not silent |
| T-03-02 | Tampering | A production-only / synthetic-injection branch corrupting the detection guarantee | mitigate | Anti-backdoor grep test asserts no `if fixture`/`if test` branch in `src/brm/`; fixtures are ordinary stored snapshots / replayed HTTP responses (D-04) |
| T-03-03 | Denial of Service | A pathologically large diff consuming memory | accept | `difflib` is stdlib and bounded by snapshot size; one curated low-volume source in Phase 1 makes this negligible; revisit if multi-source volume grows |
</threat_model>

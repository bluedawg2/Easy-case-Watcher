## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Windows ProactorEventLoop incompatible with psycopg 3 async**
- **Found during:** Task 1 verification (alembic upgrade head)
- **Issue:** Windows defaults to ProactorEventLoop; psycopg 3 async requires SelectorEventLoop. The error appears both in alembic/env.py's `asyncio.run()` call and in pytest-asyncio's test runner.
- **Fix:** (a) alembic/env.py: pass `loop_factory=lambda: asyncio.SelectorEventLoop(selectors.SelectSelector())` to `asyncio.run()` on `sys.platform == "win32"`. (b) tests/conftest.py: add `event_loop_policy` fixture returning a `SelectSelectorPolicy` (subclass of DefaultEventLoopPolicy) that overrides `new_event_loop()` to return a SelectorEventLoop.
- **Files modified:** `alembic/env.py`, `tests/conftest.py`
- **Verification:** `uv run alembic upgrade head` exits 0; `uv run pytest tests/test_models.py tests/test_lifecycle.py -x -q` 29/29 passed
- **Committed in:** `5378e1a` (Task 2 commit)

**2. [Rule 1 - Bug] Test DB URL substitution replaced username instead of database name**
- **Found during:** Task 2 verification (test run against brm_test)
- **Issue:** `settings.database_url.replace("/brm", "/brm_test", 1)` matched the first `/brm` which occurs in `//brm:brm@` (the username), producing `postgresql+psycopg://brm_test:brm@localhost:5432/brm` — wrong username in the URL.
- **Fix:** Changed to `settings.database_url.rstrip("/").rsplit("/", 1)[0] + "/brm_test"` which replaces only the last path component (the database name).
- **Files modified:** `tests/conftest.py`
- **Verification:** 29/29 tests pass with correct URL `postgresql+psycopg://brm:brm@localhost:5432/brm_test`
- **Committed in:** `5378e1a` (Task 2 commit)

**3. [Rule 3 - Blocking] Docker Desktop not installed — used PostgreSQL portable binaries**
- **Found during:** Task 2 verification (docker compose up -d db)
- **Issue:** `docker` command not found; Docker Desktop is not installed on this machine. The plan specifies Docker as the Postgres delivery mechanism.
- **Fix:** Downloaded PostgreSQL 16.3 portable zip binaries via winget/7-Zip, initialized a cluster with `initdb`, started postgres via `pg_ctl`, and manually created the `brm` and `brm_test` databases. The `docker-compose.yml` and `scripts/init-test-db.sql` remain committed as the canonical mechanism for Docker-enabled environments (and will be used by the human developer running the Docker verification in Task 3).
- **Files modified:** None (only local runtime state)
- **Verification:** `uv run alembic upgrade head` succeeds against both databases; 29 tests pass
- **Committed in:** n/a (local environment only — developer will use Docker)

<objective>
Complete the district coverage vertical slice: add index/listing mode to `PdfSourceAdapter` (D-16/D-17/D-18), wire the `run_pdf_ingest` dispatch so every real PDF-source poll stamps `pdf_provenance` (D-11), onboard the ~3-district tranche as config-only registry rows (SRC-03), and prove criterion 5 — a real PDF-based district rule change flowing end-to-end into the review-queue pipeline.

Purpose: District courts publish two structurally different ways. Local rules are one canonical PDF (document pattern — already built in 03-02). General/standing orders are an HTML index of dated order PDFs (index/listing pattern) — a new order appears as a new index entry, not an edit to an existing document. Collapsing these into one detection rule is the explicit RESEARCH anti-pattern. The provenance flag (D-11) must be stamped on real pipeline runs, not only the test path — so the dispatch from the admin/seed ingest entry point to `run_pdf_ingest` is an explicit task here.
Output: index/listing mode in `pdf.py`, the `run_pdf_ingest` dispatch wired into the admin ingest entry point, district seed rows (including CACB's canonical LBR), and the criterion-5 end-to-end replay test.
</objective>

<objective>
Deliver the thinnest end-to-end PDF document slice: PDF-specific text normalization plus the `PdfSourceAdapter` in document mode. After this plan, a real clean text-layer PDF source fetches, extracts, normalizes, hashes, and resolves CHANGED/UNCHANGED/FETCH_FAILED through the existing `run_ingest` pipeline exactly like the FRBP feed source — and a scanned or broken PDF resolves FETCH_FAILED with the correct reason code.

Purpose: Normalization is what makes PDF diffs trustworthy — raw PDF text carries hyphenation, line numbers, page headers, and ligature glyphs that produce diff noise between semantically identical versions (RESEARCH Pitfall 3). The adapter is the new `SourceAdapter` behind the existing seam; `run_ingest`, `detect_change`, `store_snapshot` stay PDF-agnostic.
Output: `pdf_normalize()`, `src/brm/ingest/pdf.py` (document mode — index/listing mode is plan 03-03), and the normalization + adapter tri-state tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external court site → PdfSourceAdapter | Untrusted PDF binaries fetched over HTTP cross into the adapter |
| PDF bytes → extract_pdf_text | Untrusted file content reaches the pypdfium2 native parser |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Denial of Service | malformed PDF crashing the parser | mitigate | `extract_pdf_text` (plan 03-01) wraps the parse in try/except → extraction_failed-class result; the adapter handles it as FETCH_FAILED, worker never crashes |
| T-03-02 | Denial of Service | oversized PDF exhausting memory | mitigate | `pdf.py` rejects bodies above `settings.pdf_max_bytes` with FETCH_FAILED BEFORE calling `extract_pdf_text` (Task 2) |
| T-03-05 | Spoofing / Tampering | a scanned PDF silently read as "no change" (silent-failure mode) | mitigate | Near-empty extraction always resolves FETCH_FAILED with a reason_code, never UNCHANGED (Task 2, RESEARCH Pitfall 1); `test_near_empty_not_unchanged` is the regression guard |
| T-03-06 | Tampering | court site returns a soft-404 / login wall as HTTP 200 with a non-PDF body | mitigate | A non-PDF body extracts as near-empty with no text layer → FETCH_FAILED reason_code=extraction_failed (Task 2) — never silently UNCHANGED |
</threat_model>
